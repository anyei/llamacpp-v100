## тмЬ Pending (approved roadmap, not started)

- [x] **9. GPU sampling in tensor-split mode** тАФ investigated and measured, **not implemented (no headroom)**. Two findings: (a) the server disables backend sampling whenever speculation is active (`tools/server/server-context.cpp:1848`), so it can never help the MTP prod config; (b) for nospec serving the entire CPU sampling path is free тАФ 4-conc aggregate 89.6 t/s greedy vs 89.5 t/s with full top-k/top-p/min-p chain vs 89.7 control (within noise): logits transfer + CPU sampler fully overlap GPU compute. Implementing would require a new allgather subgraph-boundary type in the meta backend (vocab-split logits hit the `handle_per_row` assert) for a measured ~0% gain.
- [x] **10. KV cache compression (TurboQuant)** тАФ resolved: **existing quantized KV already meets the goal; TurboQuant not worth implementing.** Findings (Qwen3.6-27B, tensor mode, wikitext-2):
  - Quantized KV **works in tensor-split mode** (old note saying otherwise was stale), and this tree already ships upstream PR #21038 (commit `744c0c731`): Hadamard-rotated KV quantization тАФ the same core trick as TurboQuant/QuaRot тАФ auto-enabled whenever KV is quantized (opt-out `LLAMA_ATTN_ROT_DISABLE=1`).
  - PPL @32k ctx: f16 6.2904 ┬╖ q8_0/q8_0 6.2922 (+0.03%, noise, **lossless, 2├Ч**) ┬╖ K q8_0/V q4_0 6.2993 (+0.14%, **2.67├Ч**) ┬╖ q4_0/q4_0 6.3093 (+0.30%, **4├Ч**). @8k ctx all statistically identical. temp-0 128-tok gen byte-identical to f16 for q8_0 and q4_0. tg cost тЙИ тИТ3-4%.
  - Mixed K/V types crashed in tensor mode: default builds compile no FA kernel for KтЙаV (`fattn.cu:424`), the scheduler tries CPU fallback and the meta backend asserts (`ggml-backend-meta.cpp:557`). Fixed by building with `GGML_CUDA_FA_ALL_QUANTS=ON` (added to `.devops/cuda.Dockerfile`). Known residual: unsupported-op CPU fallback in tensor mode aborts instead of erroring cleanly.
  - TurboQuant itself (arXiv 2504.19874): no official code; needs a non-block ggml type (Lloyd-Max codebooks + per-token norms + QJL residual) and codebook gathers inside the V100 FA vec kernels; would buy ~3.5 effective bits vs q4_0+rotation's 4.5 (тЙИ22% more capacity) тАФ 2026 practice (SAW-INT4, vLLM) converged on rotated token-wise INT4, which is exactly what's already in-tree.
  - **Recommended prod setting: `-ctk q8_0 -ctv q8_0`** (strictly lossless, halves KV тЖТ doubles usable context/slots); `-ctk q8_0 -ctv q4_0` if more capacity is worth +0.14% ppl.
